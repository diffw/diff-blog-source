# 内容价值评分标准（X-worthy Score 1–10）

## 起评分：5

## 加分项

| 触发条件 | 加分 |
|---|---|
| 有具体的时间、地点、数字、对话 | +3 |
| 反直觉 / 与主流意见相反的判断 | +2 |
| 职业 / 专业的实战经验（设计、AI、独立开发、管理） | +2 |
| 完整的故事弧（问题 → 行动 → 结果 → 教训） | +2 |
| 有普世钩子（亲子、职场、恐惧、野心、怀疑） | +1 |
| 有可被截图引用的金句 | +1 |
| 把过去经历（流利说、阿里云、杭州/上海/移民）连到当下 | +1 |
| 文章本身已发布在 diff-blog-source（已被作者打磨过） | +1 |

## 减分项

| 触发条件 | 减分 |
|---|---|
| 纯祷告 / 私人灵修 | −2 |
| 主要是书摘 / 别人观点的转述 | −2 |
| 大量基督教术语，无外人能理解的载体 | −1 |
| 与最近 30 天 tweets 主题高度重复 | −1 |
| 主要是情绪倾倒，没有反思或洞察 | −2 |
| 文章过短（<300 字）且无独立观点 | −1 |

## 阈值

- **≥7**：进入生成
- **5–6**：放进 log 的"hold"列表，下次合并多篇时再考虑
- **<5**：跳过

## 文章类型 → Tweet 输出格式

| 文章类型 | 中文输出 | 英文输出 |
|---|---|---|
| 反思性长文（>1500 字） | 1 single + 也许 1 thread | 1 single（取一个 angle） |
| 故事 / 叙事 | 1 thread（8–15 条） | 1 single（截取最强片段） |
| 编号清单（感恩、原则、年终回顾） | 2–4 single（每条独立） | 1–2 single |
| 语音转写草稿 | 先用 polisher skill 整理，再处理 | 不直接出，等润完 |
| 灵修笔记 | 默认跳过；除非有具体生活事件载体 | 跳过（10k 粉丝前不发信仰内容） |
| 设计 / AI / 独开技术札记 | 1–2 single | 1–2 single |
| 家庭故事 | 1 single（去身份化） | 1 single |

## 中英文配比的不对称

中英文不要互译。同一篇文章可能：
- 只产中文（涉及流利说本土故事，外国人理解门槛高）
- 只产英文（普世技术 tip，中文圈已饱和）
- 中英都产但内容不同（同一观点，中文走个人故事，英文走数据冲击）

英文输出更稀疏是 **OK 的**——英文圈对密度要求低，对差异化要求高。

## 跨文章组合

如果某周生成的 tweets 中：
- 同一 pillar 出现 ≥4 条 → 削减为 2 条
- 同一来源文章出现 ≥3 条 → 削减为 2 条
- 7 天内空缺某 pillar → 主动从 hold 列表里挑文章补上
