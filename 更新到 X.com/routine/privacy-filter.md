# 隐私过滤 — 永远不发到 X 的内容

> 当不确定时，**跳过 + log "review needed"**。让 Diff 手动决定，不要替他冒险。

## Hard Block（整篇文件直接跳过）

### 路径与标签

- 路径含 `private/` / `personal/` / `family-only/`（虽然目前应该没有）
- 文件名以 `_` 或 `.` 开头
- frontmatter 含 `tags: [private, 不公开, 私密, draft-private]`
- frontmatter 含 `published: false` 且无 `share-tweets: true`

### 内容主题

整篇含以下内容之一就 hard block：

- **家庭财务具体数字**：账户余额、年薪、税单、债务、房贷余额、特定股票仓位
- **婚姻冲突的细节**（"我们因为 X 大吵一架"）—— 反思婚姻的一般文章 OK，具体争吵不行
- **孩子的医疗诊断**（提到儿子有湿疹的事实 OK；具体药物 / 治疗方案 / 就诊医院 不行）
- **教会内部纪律 / 长老会议内容 / 牧师私下沟通**
- **未公开过的住址 / 邮编 / 学区 / 学校全名**
- **银行 / 信用卡 / SSN / 护照 / 律师事务所等敏感信息**
- **未经同意的他人故事**（朋友 X 的离婚、同事 Y 的辞职等）

## Soft Filter（处理后可发）

### 人名脱敏

- 朋友/同事不是公开人物 → 用 role 替代：「一位前同事」「一位多年好友」「教会一位姐妹」
- 明确公开的人物**保留全名**：王翌、白鸦、冯大辉（流利说、阿里云时代的同事/朋友，已多次公开提及）
- 家人 → 用关系名："妻子"、"女儿"、"儿子"、"小女儿"、"父亲"——不出现真名
- 例外：博客文章里已经出现过、且 Diff 主动用了真名的家人（女儿以琳/Linda、儿子安然、小女儿 Eather）—— 用 role 仍然是更安全的默认

### 公司名

- 现任合作机构（截至 2026-03-29）：**统称"我之前的机构"**，不指名
- 流利说、阿里云、阿里巴巴：**直接用全名**（已公开身份）
- 客户/合作公司没经过 Diff 公开过的：用"一家科技公司""一家创业公司"

### 金额

- 具体年薪/薪资数字 → 替换为"一笔可观的回报"或类似
- 房产价格 → 跳过或泛化为"美国买房"
- 投资收益数字 → 跳过具体数字，保留判断

### 地点

- 北美城市级别 OK：达拉斯、洛杉矶、旧金山
- 国内城市级别 OK：杭州、上海、长沙、徐州
- **小区名 / 街道名 / 教会全名** → 跳过
- 例外：已在公开博客里出现过的地标可保留（如"贝塔咖啡"——白鸦的咖啡馆，公开知名）

## 信仰内容的特殊规则

### 中文：高门槛

允许发布的信仰相关内容必须满足：
- 有具体生活事件作为载体（不是纯祷告 / 纯经文默想）
- 教训能被非基督徒读者理解和共鸣
- 不出现："弟兄""姐妹""神的恩典""祷告中""主带领"等高频圈内词作为主语

举例：
- ✅ "陪小女儿看蚂蚁，被她一句问话打醒。独立开发的人需要被这种问题打断。"（信仰内核 + 普世表达）
- ❌ "今天读到 XXX 章 X 节，神再次提醒我..."（纯灵修，跳过）

### 英文：10k 粉丝前完全跳过

英文圈对 faith content 极度极化。冷启动期任何信仰相关内容都跳过。
等中英合计 10k 粉丝后，再由 Diff 手动开放策略。
Routine 的判断：**英文 + 任何信仰相关 = 跳过**，记入 log。

## 灰色地带 → 默认跳过

如果你（Routine）在以下情况犹豫：
- 不确定某个细节是否过于私密
- 不确定某个名字是否能用
- 不确定某个观点是否会被误解
- 不确定某个金额/事件是否已公开

**默认跳过 + log "review needed: <具体理由>"**。Diff 比你更了解上下文，让他决定。
不要为了凑 tweet 数量而冒险。

## 应用顺序

1. 先看 frontmatter 标签 → hard block 立刻跳过
2. 扫正文关键词 → hit hard block 关键词跳过
3. 通过以上两关后，进入价值评分
4. 价值 ≥7 进入生成
5. 生成时应用 soft filter（脱敏）
6. 灰色地带触发 → mark `needs_review: true`，仍然写入 tweets.js（让 Diff 在 UI 里看到，由他决定发不发）

## Log 格式

每次跳过都要写入 `routine/logs/{date}.md`：

```markdown
## Skipped: privacy
- diff-blog-source/家庭风波.md
  reason: 涉及婚姻冲突具体细节
- notes/Drafts/我的薪资变化.md
  reason: hard block — 财务具体数字
```

让 Diff 能一眼看到哪些被屏蔽、为什么。如果他不同意 Routine 的判断，可以手动调整文件或写规则例外。
